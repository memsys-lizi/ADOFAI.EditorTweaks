ADOFAI Editor Tweaks

补齐编辑器工作流细节，提供离线谱面视频渲染。每个功能可独立开关，禁用时零性能影响。

项目概览

ADOFAI Editor Tweaks 是一个 UnityModManager Mod，面向 A Dance of Fire and Ice（冰与火之舞）。不修改游戏源码，通过 30+ 个 Harmony Patch 运行时注入。

数值右键拖动

数字输入框右键横向拖动微调，Int/Float/Tile/Vector2。

装饰编辑修复

拖动偏差修正、轴心十字修复、坐标吸附。

视频背景同步

中途 checkpoint 播放时自动校正视频时间。

偏好即时保存

官方偏好变更后立即落盘，防异常退出丢失。

快捷设置浮窗

可拖动 IMGUI 浮窗，吸附/步进/渲染入口。

离线谱面渲染

不录屏直接导出 MP4/MKV/MOV，1080p~8K。

Steam 云同步

手动上传/下载设置至 Steam Cloud。

安装与启用

安装步骤
1
安装 UnityModManager
从 Nexus Mods 下载 UnityModManager，对 A Dance of Fire and Ice 完成配置。
2
放入 Mod 文件
将 ADOFAI.EditorTweaks 文件夹放入游戏 Mods 目录。确保 Info.json、.dll、Resources/、Tools/ 结构完整。
3
启动游戏并启用
启动游戏，按 Ctrl+F10 打开 UMM 管理器，找到 ADOFAI Editor Tweaks 点击状态开关启用。
首次安装

首次启用时自动打开本用户手册。之后可在 Mod 目录 Resources/README.html 随时查阅。

所有功能默认开启，可在 UMM 设置面板按需关闭，关闭后零性能影响。

编辑器功能

以下功能在 ADOFAI 编辑器中生效，均可在 UMM 设置面板独立开关。

数值右键拖动

启用数值输入框拖动调节 默认开
EnableNumericDrag

官方编辑器数字输入框只能手输。此功能为以下类型附加 右键拖动 微调：

Int Float Tile Vector2

操作：输入框上按住鼠标右键横向拖动即实时变化。拖动时实时写回 LevelEvent，刷新装饰、背景、滑条、地板指示器等编辑器状态，松开后触发官方保存。

使用右键不影响左键点击、选择文字、手动输入等官方行为。
拖动参数
设置项默认值说明
整数每像素步进1Int/Tile 字段每像素拖动变化量
小数每像素步进0.1Float 字段每像素拖动变化量
小数最大位数3拖动后保留的小数位数

装饰编辑修复

修复 Camera/CameraAspect 装饰拖动 默认开
EnableCameraRelativeDecorationDragFix

官方拖动按世界坐标处理，但 Camera/CameraAspect 装饰的数据坐标本质是屏幕空间。直接套用拖动比例不自然，缩放/宽屏/官谱镜头下更明显。

Mod 按相机正交尺寸和宽高比计算屏幕空间增量，保证拖动与视觉一致。混选时普通装饰仍走官方逻辑。Shift 轴锁定继续有效。

修复装饰轴心显示 默认开
EnableDecorationPivotFix

修复 Camera/视差装饰轴心十字位置不准：单选时将十字放到装饰 Transform 位置；修正屏幕相对装饰的视差变换后位置；仅在编辑器环境且已启用屏幕裁剪时修正。

装饰移动吸附 0.5
DecorationMoveSnapStep

拖动后自动按步进吸附坐标。设为 0 关闭吸附。设 0.5 则坐标吸附到 0、0.5、1、1.5 等位置方便对齐。

视频背景同步

修复中途播放时视频背景延迟 默认开
EnableVideoBackgroundSyncFix

从 checkpoint 或选中地板播放时，长视频 seek 可能慢于谱面，表现为背景落后或漂移。

Mod 按 songposition_minusi - countdownOffset + vidOffset 校正。启动窗口（150帧）积极 seek，最多 12 次快速追上；运行期偏差超 0.35s 且距上次同步超 0.25s 才校正。循环/非循环、pit 自适应。

偏好即时保存

持久化官方编辑器偏好设置 默认开
PersistEditorPreferences

官方偏好有时只写内存、依赖游戏自己落盘。异常退出或切场景未触发保存时可能丢失。

Mod 不改变偏好项本身，只在变化后补一次 Persistence.generalPrefs.Save()。保存失败不阻塞官方 UI。

快捷设置浮窗

显示快捷设置浮窗 默认开
ShowEditorOverlay

编辑器内深色 IMGUI 可拖动浮窗，支持折叠。默认偏右上不挡操作。

展开显示：装饰吸附精度、小/整数拖动步进、小数位数、当前渲染规格、判定文字开关、仅渲染选中段落开关、渲染按钮。

渲染时：切换为模态进度窗，拦截编辑器/玩家输入，但保留控制器 Update 保证渲染推进。

浮窗位置和折叠状态自动保存。

谱面视频渲染

本 Mod 最大的功能。不录屏、不录 UI，从游戏场景离线导出视频。

渲染总览

支持场景：编辑器自定义谱面、scnGame 关卡、官谱及旧官谱场景。

输出格式：MP4（H.264/AAC，推荐）、MKV（Matroska）、MOV（QuickTime）。

音频编码：AAC（有损，默认）、FLAC（无损）、ALAC（无损，MP4 原生）。

分辨率：16 ~ 7680x4320，快捷预设 1080p / 2K / 4K，宽高强制偶数。

帧率：1 ~ 240fps，快捷预设 30 / 60 / 120。推荐 60。

编码：优先 NVENC GPU 硬编码，失败回退 x264 CPU 编码。

渲染模式：整首渲染 / 框选段落渲染（编辑器内选中连续砖块）。

核心技术

强制视觉时钟：渲染时覆盖 conductor 的 songposition_minusi，每帧固定步进推进。机器慢只影响等待时间，不影响视频时间轴。这是防止球抽搐的核心。

自动打击：每帧自动补打至当前视觉时间，最多 16 次/帧防追块。片段渲染不超选区终点。

画面捕获：使用官方 scrCamera 三相机链（Bgcamstatic + BGcam + camobj），AsyncGPUReadback 异步读回 RGBA/BGRA。保证官谱滤镜和视频背景一致。

音频捕获：Unity AudioRenderer 离线渲染至 WAV，由 FFmpeg mux。界面音效渲染期间被屏蔽。

内存管理：按分辨率动态限制 GPU pending（1080p 8帧 / 4K 4帧 / 8K 2帧）和 FFmpeg 队列，队列满时反压推进。

操作步骤

1
确保场景可渲染
打开编辑器加载自定义谱面，或进入 scnGame/官谱场景。浮窗状态显示"已就绪"。
2
调整参数
UMM 设置面板或浮窗中设分辨率、帧率、导出目录等。推荐默认 1080p60。
3
（可选）框选段落
如只需部分谱面，框选至少两个连续砖块，开启"仅渲染选中段落"。
4
点击渲染
浮窗或设置面板中点击"渲染视频"。进度窗显示帧数、速度、预计剩余。
5
完成或取消
渲染完成自动恢复编辑器。取消也安全恢复。成品在导出目录中。

段落渲染

默认渲染整首谱面。开启"仅渲染选中段落"后，在编辑器中框选至少两个连续砖块，渲染将从第一个选中砖块开始，到最后一个选中砖块后立即停止（不使用尾巴秒数）。

工作原理：片段启动时先等官方 checkpoint 进入实际游玩状态（PlayerControl），然后才开始捕获画面和音频。这避免了 checkpoint 静音淡入和黑场准备阶段录进成品。

输出文件名会追加范围后缀，如 SongName_f32-f64_20260529_153000.mp4。

段落渲染不能在 scnGame/官谱场景使用，因为需要编辑器框选功能。

排错指南

每次渲染在工作区目录 CurrentRender/render.log 写入诊断日志。排查问题时重点看：

日志标记含义
SONG_MOVED_BACKWARD歌曲视觉时间异常倒退，检查视觉时钟逻辑
PLAYER_FAILED自动打击失败，检查 conductor 和 controller 状态
AUTO_HIT ... FLOOR_JUMP自动打击跳过了砖块，检查球/log
AUTO_HIT_GUARD_REACHED单帧超过 16 次追打，谱面 BPM 可能过高
suppressedAsyncAdjusts被抑制的异步角度修正次数

如果 autoHits 正常、failedAutoHits=0、floorJumps=0，则自动播放和视觉时钟无异常。

球抽搐类问题先看 render.log，不要直接回滚视觉时钟或自动打击逻辑。

其他功能

Steam 云存档同步

使用 Steam Cloud 实现 Mod 设置的跨设备同步。云端数据以 JSON 存储，与游戏存档完全隔离。采用手动同步模式。

使用方式：UMM 设置面板顶部有"上传设置存档"和"下载设置存档"两个按钮。需要 Steam 客户端运行才能使用。

上传：将当前所有 Mod 设置写入 Steam Cloud。

下载：从 Steam Cloud 拉取配置覆盖本地。云端没有存档会提示。

不能同步的内容：面板折叠状态、文本框缓存、窗口位置等 UI 临时状态不同步。

非 Steam 版游戏或离线模式下云同步功能不可用，设置面板会显示提示。

常见问题

渲染中球抽搐或乱飘？

查看 CurrentRender/render.log，重点检查是否有 PLAYER_FAILED、FLOOR_JUMP 或 AUTO_HIT_GUARD_REACHED。如果日志干净，尝试恢复默认渲染设置后重试。

渲染结果没有声音？

检查游戏音量设置和 master mixer 是否静音。如果设置过自定义 FFmpeg 参数，检查 -map 1:a:0 是否正确指向音频轨道。也可尝试切换音频格式（AAC/FLAC/ALAC）。

显卡支持 NVENC 但没启用？

"自动均衡"档位下会自动检测 NVENC，不可用则回退 x264。确认 FFmpeg 包含 NVENC 支持（构建时自动下载的版本已包含），也可尝试重启游戏。

渲染到一半卡住不动？

降低分辨率或帧率重试。4K/8K 渲染有严格内存限制。也检查工作区磁盘空间是否充足。

导出视频画面上下颠倒？

Mod 使用 FFmpeg 的 -vf vflip 自动翻转（Unity readback 坐标与视频相反）。如果颠倒，说明自定义 FFmpeg 参数覆盖了翻转指令，清空"自定义 FFmpeg 合成参数"即可。

如何关闭某个功能？

在 UMM 设置面板中每个功能有独立开关。关闭后对应 Harmony Patch 不再生效，零性能影响。随时可重新开启。

渲染进度窗关不掉怎么办？

点击进度窗上的"取消"按钮。取消后编辑器会自动恢复编辑模式。如果模态窗没消失，在 UMM 中禁用再启用 Mod。

设置改乱了怎么恢复？

渲染设置有一键恢复默认按钮。其他设置在 UMM 面板中手动改回即可（默认值参考本文档各卡片中标注的 默认 值）。
