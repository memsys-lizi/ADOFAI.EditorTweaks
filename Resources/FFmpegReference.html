通用选项

控制 FFmpeg 整体行为的基础标志。

参数说明
-y覆盖输出文件，不询问。Mod 管道始终使用。
-n永不覆盖输出文件（与 -y 相反）。
-v / -loglevel日志详细级别。可选值：quiet、panic、fatal、error、warning、info、verbose、debug。Mod 使用 warning。
-hide_banner隐藏 FFmpeg 构建信息横幅。Mod 已启用。
-stats打印编码进度。默认启用。
-f <fmt>强制输入/输出格式。例如 -f rawvideo、-f mp4。Mod 使用 -f rawvideo 接收管道帧。
-i <file>输入文件。可多次指定多个输入源。

流映射

控制哪些输入流进入输出文件。合成操作的关键环节。

参数说明
-map 0:v:0取第一个输入（-i）的第一个视频流。
-map 0:a:0取第一个输入的第一个音频流。
-map 1:a:0取第二个输入的第一个音频流。
-map "[label]"取 filter_complex 输出标签对应的流。Mod 用于音频延迟处理。
-map 0取第一个输入的全部流。
-map -0:a丢弃第一个输入的所有音频流（反向映射）。
编写自定义合成参数时，必须自行包含 -map 指令，否则 FFmpeg 可能选择错误的流或导致流重复。

视频编码器

编码视频流的编解码器。根据质量 / 速度 / 兼容性需求选择。

参数 / 值硬件画质兼容性备注
-c:v copy无无损最佳直接复制流，不重新编码。最快，零画质损失。Mod 合成阶段默认。
-c:v h264_nvencNVIDIA GPU良好优秀硬件 H.264 编码器。极快。Mod 优先使用（可用时）。
-c:v libx264CPU优秀最佳软件 H.264。最佳质量 / 码率比。Mod 回退方案。
-c:v libx265CPU优秀良好H.265 / HEVC。同质量比 H.264 小约 50%。编码较慢，播放器支持较少。
-c:v hevc_nvencNVIDIA GPU良好良好硬件 HEVC 编码器。快，但文件比 libx265 大。
-c:v libvpx-vp9CPU优秀一般WebM 用的 VP9。免版税，编码很慢。
-c:v libaom-av1CPU最好较低最新的 AV1 编码。极慢，压缩率最高。播放器支持有限。
使用 -c:v copy（视频直通）时，无法更改码率、质量、分辨率或滤镜。视频流原封不动从临时文件传递。

视频码率控制

控制文件大小与视觉质量之间的平衡。

参数示例说明
-b:v-b:v 20M目标视频码率。M = Mbps，k = kbps。Mod 默认自动计算（1080p60 为 20M）。
-maxrate-maxrate 40M最高码率上限。防止码率尖峰。建议为目标码率的 2 倍。
-bufsize-bufsize 40M码率控制缓冲区大小。建议等于 maxrate 或目标码率的 2 倍。
-minrate-minrate 10M最低码率下限。极少使用。
-crf-crf 18恒定质量因子（x264/x265）。0=无损，18=视觉透明，23=默认，51=最差。替代 -b:v。
-cq-cq 18NVENC 恒定质量。量纲类似 CRF。Mod 默认 18。
-qp-qp 20固定量化参数。0=无损，数值越大越差。配合 -c:v h264_nvenc -qp 20。
-rc-rc vbrNVENC 码率控制模式：cbr、vbr、constqp。Mod 使用 vbr。
大多数 ADOFAI 导出场景，合成阶段使用 -c:v copy 即是最佳选择——视频已在第一阶段用正确设置编码完成。第二阶段重新编码会浪费时间并损失画质。

编码预设控制速度与压缩率的平衡。调节（tune）针对特定内容优化。

调节（Tunes）

参数优化目标
-tune film实拍 / 电影内容，更好保留颗粒感。
-tune animation卡通、平面色彩。适合 ADOFAI 谱面渲染。
-tune grain颗粒 / 噪点较重的内容，保留纹理颗粒。
-tune stillimage幻灯片、静态图像。
-tune fastdecode优化播放性能（禁用 CABAC、去块滤波）。
-tune zerolatency极低编码延迟，禁用前瞻，用于直播。
-tune llNVENC 低延迟。Mod「最快」模式使用。
-tune hqNVENC 高质量（两遍编码）。Mod「均衡 / 质量」模式使用。

视频滤镜

在编码过程中变换或修改视频流。简单场景用 -vf，多流场景用 -filter_complex。

滤镜示例说明
vflip-vf vflip垂直翻转。Mod 始终使用，因为 Unity 读回画面上下颠倒。
hflip-vf hflip水平翻转。
transpose-vf transpose=1旋转 90° 顺时针。2=180°，3=90° 逆时针。
scale-vf scale=1280:720缩放到 W×H。用 -1 保持比例：scale=1280:-1。
crop-vf crop=w:h:x:y从画面裁剪指定区域。
fps-vf fps=30通过丢 / 补帧改变帧率。
setpts-vf setpts=0.5*PTS加速（0.5× PTS = 2× 速度）或减速视频。
pad-vf pad=1920:1080:ow-iw添加边距。用 (ow-iw)/2 实现居中。
drawtext-vf drawtext=text='Hello':x=10:y=10:fontcolor=white在帧上叠加文字。需要 libfreetype 支持。
hue-vf hue=s=0调整饱和度。s=0 = 灰度。
eq-vf eq=brightness=0.1:contrast=1.2调整亮度、对比度、伽马、饱和度。
fade-vf fade=in:0:30从第 0 帧起淡入 30 帧。fade=out:300:30 为淡出。

格式位 / 像素Chroma兼容性备注

yuv422p164:2:2较高更高颜色精度。广播电视使用。
yuv444p244:4:4一般全色度分辨率。文件更大，非所有播放器支持。

bgra324:4:4较低BGRA 变体。Mod 实验性回读格式。
nv12124:2:0良好硬件编码器原生格式。高效 GPU 传输。

音频编码器

编码音频流的编解码器。

参数是否无损兼容性备注
-c:a aac有损通用高级音频编码。MP4 最佳兼容性。Mod 默认 320kbps。
-c:a libmp3lame有损通用MP3 编码器。较旧但兼容性极广。
-c:a libopus有损良好Opus 编码。任意比特率下质量最佳。兼容 MKV/WebM，不支持 MP4。
-c:a flac无损良好自由无损音频编码。支持 MKV/MP4。Mod 选项。
-c:a alac无损良好Apple 无损。MP4/MOV 原生支持。Mod 选项。
-c:a pcm_s16le无损一般未压缩 16 位 PCM（WAV）。文件极大。兼容 MOV/MKV。
-c:a copy直通最佳直接复制流，不重新编码。Mod 无法使用此方式（因为源为 WAV/PCM）。
-c:a vorbis有损良好OGG Vorbis。兼容 MKV/WebM。

音频比特率与质量

参数示例说明
-b:a-b:a 320k音频比特率。常见值：128k、192k、256k、320k。Mod AAC 默认 320k。
-q:a-q:a 2可变质量（VBR）。数字越小质量越高。libmp3lame 用 0-9（2≈190k），libopus 用 0-10。
-compression_level-compression_level 8FLAC 压缩级别：0=最快/最大，12=最慢/最小。默认 5。
-ar-ar 48000音频采样率（Hz）。可选 44100、48000。Mod 默认保持源采样率。
ADOFAI 谱面音频，320kbps AAC 对大多数听众已透明无差别。FLAC/ALAC 无损格式可完美保留原始 WAV 质量，但文件会大很多。

MP4 / MOV 标记

-movflags 选项控制 MP4/MOV 容器行为。

参数效果
+faststart将 moov atom 移到文件开头。支持下载完成前开始播放（Web 流式）。Mod 默认。
+frag_keyframe在每个关键帧处开始新分片。适合流式传输。
+empty_moov写入初始空 moov atom。用于直播流。
+separate_moof每个分片写入独立 moof atom。
+delay_moov延迟到文件末尾再写 moov。faststart 的反向操作。
+write_colr写入颜色信息 atom。HDR 内容必备。

ADOFAI Mod FFmpeg 流水线

编写自定义参数前，理解两阶段流水线至关重要。

阶段一 — 视频编码（自动处理）

ffmpeg -y -f rawvideo -pixel_format rgba -video_size 1920x1080 -framerate 60 \
       -i - \                                    # stdin: GPU 输出的原始 RGBA 帧
       -an -vf vflip \                           # 无音频；翻转 Unity 的上下颠倒画面
       <encoder> \                                # h264_nvenc / libx264 + 预设 + 码率
       -pix_fmt yuv420p \                        # 通用像素格式
       temp_video.mp4

阶段二 — 音频合成（可通过专业设置自定义）

ffmpeg -y \
       -i temp_video.mp4 \                       # 阶段一输出
       -i audio.wav \                            # Unity AudioRenderer 采集
       [filter_complex for audio delay] \        # 音频同步偏移时自动处理
       <自定义参数> \             # 自定义合成参数字段
       final_video.mp4                           # 输出文件（格式按所选扩展名）

Mod 自动生成的阶段二参数为：

-map 0:v:0 -map 1:a:0 -c:v copy -c:a aac -b:a 320k -ac 2 -movflags +faststart

自定义合成参数 — 字段说明

在专业设置中填写「自定义 FFmpeg 合成参数」字段后，Mod 会：

  1. 仍自动处理：输入文件（-i temp_video.mp4 -i audio.wav）、音频同步偏移的 filter_complex，以及输出文件路径。
  2. 完全接管：-map、-c:v、-c:a、-movflags 及其他所有输出参数。
编写自定义参数时，必须自行包含 -map 指令，否则 FFmpeg 可能无法正确连接流。最低要求： -map 0:v:0 -map 1:a:0。

字段格式

字段接受单个字符串的 FFmpeg 输出参数。所有参数作为一个字符串写入，例如：-map 0:v:0 -map 1:a:0 -c:v copy -c:a flac

可以做什么

  • 更改音频编码和参数：-c:a libmp3lame -b:a 192k、-c:a flac -compression_level 12
  • 重新编码视频（不建议——会拖慢最终合成）：-c:v libx264 -preset slow -crf 18
  • 按格式更改容器标记：MKV 可去除 -movflags；MOV 可使用 -movflags +faststart
  • 添加元数据：-metadata title="我的 ADOFAI 谱面" -metadata artist="作者名"
  • 应用音频滤镜：使用自定义 filter_complex 前缀
  • 输出多路音频流：使用 -map 包含多个音轨

不能做什么（字段限制）

  • 更改输入文件（自动管理）
  • 更改音频同步偏移处理（自动管理）
  • 更改输出文件路径（自动管理）
  • 为编码阶段添加视频滤镜（那是阶段一；请使用编码档位设置）

实用配方示例

复制粘贴到自定义合成参数字段即可使用。

基础：等效默认（AAC 320k 立体声，MP4）

-map 0:v:0 -map 1:a:0 -c:v copy -c:a aac -b:a 320k -ac 2 -movflags +faststart

无损音频：MP4 中嵌入 FLAC

-map 0:v:0 -map 1:a:0 -c:v copy -c:a flac -movflags +faststart

无损音频：MP4 中嵌入 ALAC（兼容苹果）

-map 0:v:0 -map 1:a:0 -c:v copy -c:a alac -movflags +faststart

体积优先：AAC 192k 立体声

-map 0:v:0 -map 1:a:0 -c:v copy -c:a aac -b:a 192k -ac 2 -movflags +faststart

兼容老播放器：MP3 320k 音频

-map 0:v:0 -map 1:a:0 -c:v copy -c:a libmp3lame -b:a 320k -ac 2 -movflags +faststart

FLAC 最高压缩

-map 0:v:0 -map 1:a:0 -c:v copy -c:a flac -compression_level 12 -movflags +faststart

添加元数据标签

-map 0:v:0 -map 1:a:0 -c:v copy -c:a aac -b:a 320k -ac 2 -movflags +faststart -metadata title="我的 ADOFAI 谱面" -metadata artist="作者名"

重编码视频降低码率（慢但文件更小）

-map 0:v:0 -map 1:a:0 -c:v libx264 -preset veryfast -crf 23 -c:a aac -b:a 192k -ac 2 -movflags +faststart

音频响度归一化

-map 0:v:0 -af loudnorm=I=-16:LRA=11:TP=-1.5 -c:v copy -c:a aac -b:a 320k -ac 2 -movflags +faststart

立体声转单声道

-map 0:v:0 -af "pan=mono|c0=0.5*c0+0.5*c1" -c:v copy -c:a aac -b:a 160k -ac 1 -movflags +faststart

MKV 容器 + FLAC + 元数据

-map 0:v:0 -map 1:a:0 -c:v copy -c:a flac -compression_level 8 -metadata title="谱面名"

MKV 中 Opus 音频（最佳质量比特率比）

-map 0:v:0 -map 1:a:0 -c:v copy -c:a libopus -b:a 192k -ac 2
提示：如果输出格式改为 MKV，记得在基础设置中把「视频输出格式」也改为 MKV，使文件扩展名匹配！
